## Metadata
**Name:** `terraform/workload_host_port_not_specified`
**Id:** `4e74cf4f-ff65-4c1a-885c-67ab608206ce`
**Cloud Provider:** terraform
**Severity:** Low
**Category:** Networking and Firewall
## Description
Verifies if Kubernetes workload's host port is specified

#### Learn More

 - [Provider Reference](https://registry.terraform.io/providers/hashicorp/kubernetes/latest/docs/resources/pod#host_port)

## Non-Compliant Code Examples
```kubernetes
resource "kubernetes_pod" "test" {
  metadata {
    name = "terraform-example"
  }

  spec {
    container {
      image = "nginx:1.7.9"
      name  = "example"

      env {
        name  = "environment"
        value = "test"
      }

      port {
        container_port = 8080
      }

      liveness_probe {
        http_get {
          path = "/nginx_status"
          port = 80

          http_header {
            name  = "X-Custom-Header"
            value = "Awesome"
          }
        }

        initial_delay_seconds = 3
        period_seconds        = 3
      }
    }

    dns_config {
      nameservers = ["1.1.1.1", "8.8.8.8", "9.9.9.9"]
      searches    = ["example.com"]

      option {
        name  = "ndots"
        value = 1
      }

      option {
        name = "use-vc"
      }
    }

    dns_policy = "None"
  }
}

```

## Compliant Code Examples
```kubernetes

  metadata {
    name = "terraform-example"
  }

  spec {
    container {
      image = "nginx:1.7.9"

      env {
        name  = "environment"
```
